# Контекст проекта Nixconf для Gemini CLI

Этот файл `GEMINI.md` предоставляет инструкции и контекст для взаимодействия с Gemini CLI в рамках проекта Nixconf.

## 1. Обзор проекта

Проект Nixconf предназначен для управления конфигурациями NixOS и Home Manager. Он содержит следующие основные компоненты:
- **desktop.nix**: Конфигурация для рабочей среды NixOS.
- **home.nix**: Конфигурация для Home Manager, определяющая пользовательские пакеты и настройки.
- **overlays.nix**: Пользовательские оверлеи для пакетов Nix.
- **packages.nix**: Дополнительные пакеты Nix.
- **services.nix**: Определения системных служб.
- **settings.nix**: Общие настройки системы.
- **castom/**: Пользовательские конфигурации (например, gns3, hugo).
- **configs/**: Различные файлы конфигурации (например, .bashrc, .tmux.conf).
- **i3/**: Конфигурация оконного менеджера i3.
- **polybar/**: Конфигурация и скрипты для Polybar.

Основная цель проекта — обеспечить воспроизводимую и декларативную настройку моей системы.

## 2. Стиль кодирования и рекомендации

Пожалуйста, придерживайтесь следующих рекомендаций при работе с файлами Nix:

- **Отступы**: Используйте 2 пробела для отступов.
- **Форматирование**: Старайтесь следовать общепринятому стилю Nix-кода.
- **Комментарии**: Добавляйте комментарии для объяснения сложных секций или нетривиальных решений. Комментарии должны быть на русском языке.
- **Именование**: Используйте camelCase для переменных и snake_case для имен файлов и директорий, если это не противоречит стандартным практикам NixOS.

## 3. Общие инструкции для Gemini CLI

- **Язык**: Все ответы и генерируемый код должны быть, по возможности, на русском языке, если это не противоречит синтаксису кода.
- **Приоритет**: При внесении изменений всегда отдавайте приоритет существующим конвенциям и структуре проекта.
- **Верификация**: После внесения изменений, пожалуйста, предложите шаги для верификации (например, `nixos-rebuild switch` или `home-manager switch`).
- **Вопросы**: Если есть неясности, всегда задавайте уточняющие вопросы.

## 4. Пример типового взаимодействия

**Пользователь**: "Добавь пакет `neovim` в `home.nix`."

**Ожидаемые действия Gemini**:
1. Прочитать `home.nix`.
2. Найти секцию `home.packages` или аналогичную.
3. Добавить `pkgs.neovim;` в список пакетов, соблюдая существующий стиль.
4. Предложить выполнить `home-manager switch` для применения изменений.
